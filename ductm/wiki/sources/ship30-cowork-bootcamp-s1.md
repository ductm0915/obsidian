# Claude Co-work Bootcamp 2.0 — Session 1: FileMaster

Live session taught by [[mitch-harris]] (AI wizard) and [[nicholas-cole]] (co-founder [[ship-30-for-30]]). ~58 min bootcamp. Covers foundations of Claude Co-work and the FileMaster plugin for workspace management.

## Key Claims

1. **Claude Co-work is the "middle harness"** — sits between Claude Chat (conversational) and Claude Code (full-stack coding). Designed specifically for getting work done — productivity, business tasks, writing automation.
2. **A harness wraps around the AI model** giving it rules, tools, and the ability to act on your behalf. Co-work is one such harness; others include Cursor, Windsurf, Open Claw, Claude Code.
3. **Skills = muscle memory for AI.** They are reusable prompt bundles that get triggered by context, chain together ("flick a domino"), and let you step away while they run.
4. **FileMaster** is the flagship plugin of the bootcamp: 4 skills bundled together for workspace management.
5. **Context management is the #1 skill** for working with AI — telling the agent where to look and what to remember.
6. **The co-worker folder** (mounted locally) is the agent's brain: config, index, activity log, organized subfolders. Enables persistent memory across sessions.
7. **"Tokens in, revenue out"** — Mitch's mantra: if you're doing anything someone pays you for, have AI help.

| Interface | Role |
|---|---|
| **Chat** | Conversational; can create artifacts, write code snippets |
| **Co-work** | Middle harness; productivity + work automation; uses plugins/skills |
| **Code** | Full-stack development; build apps, databases, automations |

---

## FileMaster Plugin — 4 Skills

| Skill | What It Does |
|---|---|
| **Organize** | Project-based file organization of local folders (downloads, documents) |
| **Ingest** | Feeds context/files into the agent's knowledge base; updates config + index + activity log |
| **Search** | Find anything within the co-worker workspace or mounted folders |
| **Status** | Check agent's self-understanding of the job; validate and update context |

## Co-worker Folder Structure

Downloaded as a zip, mounted as local folder in Co-work:
- `claude.md` — Identity, rules, tools for the agent
- `output/` — Where agent puts generated files
- `projects/` — Active project contexts
- `training/` — Training materials / context files
- `prompts/` — Saved prompt patterns
- `voice/` — Voice samples and writing style configs
- `references/` — Reference material
- `archive/` — Completed/old work
- `inbox/` — New files to be processed

## Model Recommendation (Co-work specific)

> *"Sonnet is much cheaper and almost as good as Opus. Don't use Haiku — it's not very good in Co-work."* — [[mitch-harris]]

- **Sonnet**: Recommended for everyday tasks (co-work default)
- **Opus**: Novel problem-solving, stretching limits
- **Haiku**: Not recommended for Co-work
