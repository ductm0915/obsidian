# Prep the Page

**Prep the Page** is [[nicholas-cole]]'s writing framework for eliminating writer's block: segment the document into labeled sections *before* writing a single word. You fill a skeleton instead of building from a blank page.

## The Problem It Solves

Writer's block comes from cognitive overload — holding headline, intro structure, main points, and conclusion in mind simultaneously. Each element requires different thinking. Doing them all at once creates paralysis.

Prepping the page separates *structural decisions* from *writing* — two distinct modes that shouldn't happen at the same time.

## The Process

1. **Working title first** — sets direction and often encodes the structure ("3 Reasons" = 3 sections)
2. **Intro skeleton** — label 4 slots: what it's about / who it's for / the promise / credibility
3. **Main point headers** — one header per reason/tip/mistake; these are the highest-value decisions
4. **Conclusion slot** (optional) — only if you have something new to say; otherwise use [[the-cliff]]

## Why It Works

Once the skeleton exists, the document no longer looks blank — it looks *almost finished*. This lowers the psychological barrier to starting and creates momentum.

> "At a glance you almost don't even realize it's not fully filled in — you go 'oh I'm already halfway done.'"

The structure also forces clarity on what you're promising the reader before you write — which makes the writing more focused and the finished piece more skimmable.

## Readability = Skimmability

Readers make a 5-second decision: headline → intro → main point headers. If the structure is scannable, they stay. Prepping the page builds skimmability into the document from the start rather than retrofitting it during editing.
