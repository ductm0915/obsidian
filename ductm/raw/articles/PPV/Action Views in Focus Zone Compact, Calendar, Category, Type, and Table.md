### The Difference Between C1 and C2

So the difference between the uncatategorized list which is C1 and the categorized list C2 is rooted in function. The uncatategorized list, this is one flat list of actions. Everything is in one stream. This is a single execution list, one cube.

The categorized list, this is the same underlying list of actions grouped by category. It adds a layer of context balance. So you can see how my actions split between these categories.

That grouping gives you a few advantages. Number one is mode concentration and mode switching. So when you're staying in one mode, for instance, work mode, you keep your attention inside the work group section and the work action queue, it helps you hold concentration inside work while ignoring everything else. When you want to change mode, say, I'm ready for a lighter personal action. You don't need to scan the whole mix list. You just jump directly to the personal category group and choose from the top of that queue.

Another advantage is category aware capture. So when you add a new action inside each category group, the category is set automatically. You can drop new actions straight into personal, work, or connection without the extra step of manually assigning categories.

So it's still lightweight, but now you have separate cues for personal, work, and connection. You can add new actions straight into the right queue, and you can both switch modes quickly and stay concentrated inside a single mode without scanning a long mix list. ## Category and Type Board Views

Then we have category and type board views. Now, category and type, these two views, they are great for context and batching actions. Category and type help you see your actions in the right context and maximize efficiency.

### Category View

So, the category view groups actions into personal work and connection. Each card is still one action. If you drag a card between columns, you're not creating new actions. You're just changing its category.

Seeing this view may prompt questions like, is everything about work right now or do I need enough connection? like do I have enough connection activities in the mix? Knowing which part of your life each action belongs to helps you adjust the balance between them. It also helps you choose from the right bucket depending on where your attention and energy are. So you can use this briefly when planning or reviewing the type view.

### Type View

Here we group actions by what kind of actions they are. Correspondence, errands, meetings, content, etc. Again, each card is the same action you saw before.

Type is powerful for two reasons. Number one, batching. You can run similar actions together like correspondence, meetings, errands, content. You can set a correspondent block and work through all of your calls, emails, messages, and replies in one pass. And when you're out and about, you can look at just the errand group and knock out several stops in the same trip. When similar actions are grouped, you can run them together efficiently without constantly switching the context.

The second way the type grouping can be very powerful is workload awareness. A day with many meetings or deep content items feels much heavier than a day of light errands. So the typeboard lets you see the weight of your day at a glance so you're not frequently surprised by how demanding the day is.

So these are organization lenses you use when you want to understand and adjust how your actions are shaped within the type view. There are a few types worth calling out. We'll dive into meetings in the meetings tab and routines in the routine section which is coming up later.

## List SOP: A Special Action Type

And here I want to zoom in on one specific type and that is list SOP. List SOP is for actions that contain a list of sub actions or a batch set of actions inside a page. This type of action always happens as a bundle. The list SOP label tells you that there's a list of steps in here. Please open this page.

You can use it for personal things like grocery run, packing list, weekly house cleaning and resets. You could also use it to carry out a repeatable SOP and that is standard operating procedure that you will reuse like client on boarding checklist or content publishing workflow.

In all of these cases, the page is not just notes. It's an actionable list you intend to run through. When you see this type, don't just check off the action itself and move on. Open the page and run the internal checklist. Check off each step as you go. After you run through the steps on the checklist, mark the whole action as done.
