## The Value of Source Views

Another point I would like to illustrate regarding views is the value of source views. Now to access source views, we are not looking at these embedded views in alignment zone. We need to go to the navigation and data source. Go to GPR database.

And every view you see here, these are source views because they live in the source GPR database. Source views are view tabs built in the source database. When you're working with GPR views, it's important to remember that the source database is the home base.

The views that are built in with the source GPR database are already designed to cover the most common use cases out of the box. So start by using the existing views in the source GPR database as much as you can.

### Available Source Views

So we have the master view which is something that is similar to the alignment zone view and we have compact. This one is organized by categories again and you get to see the status as well and the dates. So this is highlighting category grouping but in a compact way.

Project status. Use the autodata information to help you adjust the date and status and priority.

And routine status, you get to see what routines are currently active and which ones you have planned but not started yet. So you can start making them active and start creating actions for them.

So these source views cover most common use cases.

## Guidelines for Working with Source Views

So the first guideline regarding source views is use and adapt the existing source views whenever possible because these are already built for you and we have spent a lot of time configurating the views, the property options, the filters and the sort. So this minimize the amount of work you need to do. You just embed these existing views wherever you need them.

Then when you need a new view that you expect to use very often, it's almost always better to create it directly in the source database as a source view, which is where we are right now. Then embed that source view, that new source view wherever you need it.

You configure your own filters, sorts, and layout once in the source. After that, you can drop that same view in any other pages without redoing filters and layout from scratch for each new embed. In other words, one time set up many reuses.

When you do need something more specific and expect to use it often, create the new view in the source database first, then embed that source view wherever it is most useful in your workflow. That way, you design it once, reuse it in multiple places, and keep everything easy to maintain.
